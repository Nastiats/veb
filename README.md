# лабораторная работа 1  проектирование архитектуры динамического веб-приложения

Тема: разработка сайта по продаже железнодорожных билетов.

Разработка системы электронной продажи билетов на поезда с использованием личного кабинета. Система позволяет регистрировать поезда, рейсы, количества мест на них. Также, осуществляет продажу билетов пассажирам.

Целевая аудитория: пассажиры железной дороги.

Проект будет иметь несколько режимов работы: пользовательский и системный. 

В системном режиме система должна позволять регистрировать поезда, связанные с ними рейсы, количество в каждом поезде вагонов разного типа, получать общий отчёт о количестве проданных билетов. 

В пользовательском режиме система должна отображать расписание отправления поездов, количество свободных мест различного типа, возможность их резервирования (приобретения), в личном кабинете должна быть возможность отображения приобретённых пользователем билетов.

Структура приложения:

Главная (/) – общая информация, возможность вызова странички аутенфикации;

Аутенфикация 
* ввод логина и пароля (/auth), 
* ссылка на станицы восстановления пароля (/auth/restore), 
* регистрации нового пользователя (/auth/new);

Работа в личном кабинете: 
* Выбор поезда (номер, дата отправления, тип вагона) (/select); 
* Покупка билетов на выбранный поезд (/sale); 
* Просмотр списка приобретенных билетов с возможностью повторить приобретение билетов на другую дату (/mytickets);

Администрирование (/adm) – состоит из нескольких разделов: 
* Аутентификация (/adm/auth); 
* Ввод типа вагона (/adm/vtype); 
* Формирование поезда (/adm/train); 
* Формирование расписания движения поездов (/adm/schedule); 
* Получение статистики проданных билетов (/adm/stat);

Технологический стек

* На стороне браузера выполняется javascript приложение с использованием фреймворка react.js ;
* На стороне сервера выполняется приложение написанное на языке JAVA с использованием фреймворка Spring;
* Для хранения данных используется СУБД PostgresSql.
